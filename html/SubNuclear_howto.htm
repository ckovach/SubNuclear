I. Introduction and overview

SubNuclear is a matlab toolbox designed for two uses. The first is to implement a semi-automated protocol for superimposing atlas-derived substructures on subcortical structures visible in MR's.  The second is to provide a simple and adaptable set of matlab tools for exploring medical images and localizing electrode positions based on imaging artifacts.  To those ends, the toolbox allows one to view multiple planes of image volumes and superimpose the outlines of 3D mesh objects, to select points within the volume, to edit meshes, to apply thin plate spline (TPS) warping in 3 dimensions using manually selected control points and to compare the same views across multiple images.  

The coregistration protocol takes 3  inputs: (1) structural MRs for individual subjects, which include a preoperative image and a postoperative (i.e. post-implantation) image, with the latter containing electrode-related magnetic susceptibility artifacts; (2) atlas-derived anatomical features: the toolbox currently includes 3D meshes for the major amygdala subnuclei, which were constructed from the plates of the ÒAtlas of the Human Brain, 3rd Ed.Ó (Mai et al., 2008); and (3) Parcellated subcortical structures obtained with FSL's (v5.0) automated image processing, coregistration and parcellation method as implemented in fsl_anat, which coregisters structures in the preoperative image with template meshes representing gross subcortical structures {Patenaude, 2011 #3830}. This requires FSL to be installed.

Because automated parcellation algorithms are less accurate than hand tracing, the result of segmentation should be manually adjusted to conform more closely to the subjectÕs anatomy. The atlas-derived structures and the templates must in turn also be coregistered with each other. For the amygdala structures included in the toolbox, this step has already been done by aligning the principle axes of the mesh surfaces given by a singular value decomposition, matching second-order statistics between atlas-derived and template meshes through rotation and scaling, followed by projection of the vertices of the template meshes onto the atlas-derived structures. Meshes representing the subnuclei were then transformed from the atlas space to the template and subject space using TPS warping, for which the vertices of respective meshes served as control points.

Much of the protocol is automated, but it requires close supervision and at various steps manual intervention. The most time-consuming part is likely to be the coregistration of pre- and post-operative images, as this is best done using TPS warping with manually selected control points. This step is important because displacement of the brain and postoperative swelling and other sources of image distortion lead to significant structural differences pre vs. post. Linear and automated non-linear coregistration generally donÕt suffice to bring the images into alignment so warping with manually chosen control points is the best option.

III. Viewing volumes

 

SubNuclear uses MatlabÕs object oriented programming capabilities, implementing a number of custom classes.  The most important of these is the volumeview class.  To open a volume one must instantiate a member of the class with the command.

            >> vol = volumeview([nifti file ending in .nii or .nii.gz])

If no argument is given, youÕll be prompted to select a file. The main GUI window will appear with axial, coronal and sagittal views of the selected volumes.

 

Figure 1.  Volumeview GUI

 

 

The views can be manipulated using the standard options in the matlab figure menu bar, which allow zooming and panning. The plane of the view can be changed with the scroll-bar to the right of the corresponding axis, by clicking on a point in either of the other two axes, or by entering coordinates in the coordinate boxes. The green crosshairs show the intersection of the other axes and can be turned on or off with the radio button labeled Ò+Ó.

 To adjust the brightness and contrast, change the intensity_range property of the object. For example,

                       >>vol.intensity_range  = [0 255];

adjusts the color scale so that 0 or less is black and 255 or greater is white.

1.Using the GUI

An overview of the graphical user interface is shown in Figure 1. It should be self-explanatory for the most part. Objects related to an image include points, meshes and transforms, which are selected using the list boxes on the right. For any selected object annotations can be viewed and edited within the annotation window on the left. The visibility of mesh and point objects in the axis window is toggled using the ÒToggle hideÓ option in the context menu called up by right-clicking in the respective listbox.

a. Points

Coordinates of the currently selected point (center of cross-hairs) are displayed in both voxel units and in units under the selected transform in the transform list box.  The current point is added to the list of points with the ÒAddÓ button.   

b. Meshes

Mesh objects can be loaded from vtk files using both the ÒLoadÓ and ÒAddÓ options in the listbox menu. They can be loaded from the matlab workspace using the addmesh method (see below) at the command line. Mesh objects can be edited by enabling the ÒAdjust MeshÓ radio button. Further details are in the following sections.

c. Slice

The ÒSliceÓ option in the mesh and point context menus creates a new axis window with a view plane of arbitrary orientation. The orientation of the plane is determined from the first two dimensions of a singular value decomposition applied to selected points or meshes. To create a view-plane of arbitrary orientation add at least 3 points within the desired plane to the list of points, select the points and choose ÒSliceÓ. The new view will open in a separate figure.

            d. Crosshairs

Crosshairs showing the current point can be turned on or off with the Ò+Ó radio button. The location and width of each line shows the line of intersection of all other current planes of view.

 

2.Working with volumeview objects

a. Passing by reference

Volumeview objects are handle class objects, which are passed by reference, meaning that if you let

               >> vol2 = vol

              >> vol3 = vol2

vol, vol2 and vol3 now all refer to the same object, so that any change to vol2 will also be reflected in vol3 and vol. If you want to make a copy of an object so that any change of one a leaves the others unaffected, then you must create a new instance of the class by passing it as an argument to the volumeview constructor.

                       >> vol2 = volumeview(vol)

b. Loading and saving

            Volumeview classes can be saved to and loaded from matlab files, however to get the GUI going you need to reinitialize the object after loading by passing it as an argument to volumeview:

                        >> load saved_data  vol

                       >> vol = volumeview(vol)

                      c. Methods

The following functions associated with the volumeview class can be used at the matlab command line

 

vol.addmesh() :  Add a mesh to the current volume

vol.addpoint() :  Add a point to the current volume

vol.addtransform() :  add a transform

vol.resetaxis()  : reset axes to their original state. Try this if the GUI becomes unresponsive.

vol.plotupdate():  update the plot.

d. Properties

            The following data can be accessed through the volumeview object

vol.volumes   :    Medical image class

  vol.volumes.Data:   3-d array containing the image data

vol.points  :   array of point objects

vol.meshes : array of mesh objects

vol.transforms:  array of transform objects

 vol.intensity_range:  Range used in plotting the volumes.

  vol.sisters :  Array of sister volumeview objects. To make two volumeview objects sisters,  use             the command

                          >>vol1.sisters = vol2

              Sister objects can be linked so that their axis views remain the same. This is especially useful for comparing two volumes.

vol.current.(object type):  An array of the most recently selected objects of type object type, which can be ÔmeshÕ, ÔpointÕ, ÔtransformÕ, etc.

         vol.current_object: An array of the most recently selected object(s).

 

 

IV. Subcortical parcellation

 

For a gross anatomical parcellation of subjectsÕ preoperative images, SubNuclear calls FSL's module, FIRST, which generates among other things a set of meshes representing the boundaries of major subcortical structures. These are then coregistered with atlas-derived meshes, which contain finer-scale structures.  The full sequence of events is implemented in the matlab script, do_registration.  Each section of the script should be run in sequence, not all at once, i.e. don't run the command do_registration, but evaluate each section delimited by ' %%' in turn.

do_registration first requests the subject ID . All data for a given subject will be placed in the folder [subject_id]fsl.anat.

 

A. FSL

 

Subcortical parcellation uses the image processing sequence implemented in the script fsl_anat. The FSL module, FIRST, provides a gross parcellation of all the major subcortical structures including amygdala. Because the accuracy of current automated methods is in general much less than that achieved by hand, these parcellations should be checked and if necessary manually corrected.  The fsl scripts are self-contained and are generated and run with the matlab command:

            >> run_fsl_script

You will be prompted to give the preop and postop MR images, which for members of our lab can usually be found on the  HBRL data server  in the .../Images/Image_processing subfolder of a given subject's data folder. Running the complete script takes about an hour on our computational server. While this is running you can begin to choose control points for the warping between preop. and postop. brains (next section).  

The result of the script will be:

1.     The original preop. MR in standard orientation ( [sid]fsl.anat/T1.nii.gz)

2.     Additional volumes with the original image linearly and non-linearly  coregistered with the MNI152 standard brain, as well as transforms and displacement fields.

3.     Volumes with labels for parcellated structures.

4.     Meshes for each extracted structure, contained in .vtk files.   

 

B. Loading Meshes

FSL creates surface meshes representing the boundaries of parcellated structures saved as vtk files. These meshes can be loaded into the space of the current volume by right clicking on the mesh list and selecting ÒAddÓ or by invoking vol.addmesh at the command line. This will allow you to select a .vtk file, which contains the mesh data in a standard format.  Matlab TriRep objects can also be imported using vol.addmesh(TR), where TR is a TriRep object. After the FSL parcellation is complete, the next step in do_registration loads the left and right amygdala and hippocampus  meshes.

 

1. Editing Meshes

To manually edit the amygdala meshes, first create a copy of the mesh you want to edit (right click and select ÒCopyÓ) then make sure the ÒAdjust meshÓ radio button is enabled. Select the mesh in the mesh list window. Meshes can be edited in three ways:

1.     Translation within the plane of a view. To do this click on the mesh in one of the axes and drag it.

2.     Expanding the mesh near the current point. Clicking on ÒExpandÓ expands the mesh in the direction normal to the vertex by an amount that depends on the distance to the current point and angle the between the difference vector and the vertex normal.

3.     ÒContractÓ has the same effect as ÒExpandÓ only in the opposite direction.

C. Atlas Coregistration for Amygdala

 

After you're satisfied with the amygdala parcellation, atlas-derived meshes for amygdalae are loaded from the file,  mai_template_mni_aligned.mat, and subnuclei are loaded from maiwarp2mni.mat. The variables, newmeshR and newmeshL contain meshes for right and left amygdalae, respectively, as TriRep objects  (see matlab help for details on the TriRep class). The vertices in newmeshL and newmeshR are matched to those in the FSL template mesh for the amygdala (Fig. 3). TPS warping to the subjectÕs voxel space is accomplished using vertices in the manually corrected FSL mesh as matching control points. The resulting warping function is then applied to the vertices of meshes for each subnucleus in order to transform the subnuclei into the subject's image space. Adjusted meshes can then be imported into volumeview using  the ÒaddmeshÓ method.

Positioning of left amygdala subnuclei (solid) after linear alignment (left panel) and after warping into template space (right panel), using the vertices of the aligned outer amygdala mesh surfaces as control points. V. Electrode localization and preop to postop warping

 

The previous steps brought meshes for the atlas-derived structures into the space of the preoperative image. In order to localize the electrodes, we identify the center of susceptibility artifacts in the postop. image. To relate the corresponding coordinates to anatomy, they have to be transformed into the preop. space.  In general, the postop. brain is significantly distorted relative to preop. due to displacement, swelling and imaging artifacts, so that automated coregistration is not applicable. TPS warping with manually selected control points is used to create a suitable non-linear transformation. Control points can be based on fixed landmarks, such as AC and PC, but can also depend on any landmarks clearly visible in both images.

Selecting control points is a matter of adding points to the list of point objects visible in the volumeview windows for the pre-op and post-op images, making sure that they match as closely as possible in all planes. This may require some trial and error. It is also important that the ordering of the control points be kept the same between images. Points can be chosen within a given region of interest, such as temporal pole, or across the whole brain, as long as a sufficient number of points surround the structures you are trying to match.  For the amygdala, I take at least the 6 points near the dorsal, ventral, mesial, lateral,  anterior and posterior edges of the amygdala and additional points marking prominent features, such as gyral confluences and features of the gray-white boundaries.

 

 

Figure 5.  Pre- and post-op image coregistration. Electrode locations (circles) are identified using the artifact in the post-op image (right) and projected into the preop image (left) using 3D thin-plate spline warping. Control points for TPS warping (+s) are selected manually to match position with respect to local anatomy.  Control points outside the plane of view are not shown.
